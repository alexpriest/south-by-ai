# South by AI v3 Design

## New Features

### 1. Map View

New view mode alongside Timeline and List. Shows scheduled sessions on an interactive map of downtown Austin.

**Implementation:** Leaflet.js (free, open-source) loaded via CDN. No npm package needed — use dynamic script loading to avoid SSR issues.

**Venue coordinates:** Static lookup table mapping ~25 major SXSW venue names to lat/lng. The SXSW API only returns venue name strings (e.g. "Austin Convention Center, Room 12A"), so we extract the root venue name and look up coordinates. Unknown venues fall back to a "Downtown Austin" center point.

**Map features:**
- Markers for each session, color-coded by track
- Click marker → popover with session details (reuse existing SessionPopover pattern)
- Filter by time slot (morning/afternoon/evening toggle)
- Show walking route between consecutive sessions (dashed line connecting markers in time order)
- Top picks get larger/prominent markers

**Files:**
- New: `lib/venue-coords.ts` — venue name → {lat, lng} lookup
- New: `components/schedule/map-view.tsx` — Leaflet map component
- Modify: `app/s/[id]/schedule-view.tsx` — add "Map" to view toggle

### 2. Location-Aware Schedule Generation

Update Claude prompts to consider venue proximity when building schedules.

**Approach:** Add venue information to the prompt with a note about minimizing travel between consecutive sessions. Don't add coordinates — just tell Claude to prefer sessions at the same or nearby venues when quality is equal.

**Files:**
- Modify: `lib/claude.ts` — add venue proximity instruction to both generate and refine prompts

### 3. Calendar Export (.ics)

Download button that generates an .ics file containing all scheduled sessions.

**Implementation:** Server-side API route that takes a schedule ID, generates iCalendar format text, returns as file download. No library needed — .ics is a simple text format.

**Each event includes:** title, start/end time (CT timezone), venue as location, AI reason as description, SXSW event URL.

**Files:**
- New: `app/api/calendar/[id]/route.ts` — generates .ics file
- Modify: `app/s/[id]/schedule-view.tsx` — add download button in header

### 4. Session Swapping

Let users click an alternative session to promote it to "Top Pick" and demote the current top pick.

**How it works:**
- In timeline view: clicking an alternative session's "swap" button makes it priority 1 and demotes the old top pick to priority 2
- In list view: same behavior via a swap icon on alternative cards
- Changes save to KV immediately via a new API endpoint
- No page reload — optimistic UI update

**Files:**
- New: `app/api/schedule/[id]/swap/route.ts` — swap endpoint
- Modify: `components/schedule/timeline-block.tsx` — add swap button for alternatives
- Modify: `components/schedule/session-card.tsx` — add swap button for alternatives
- Modify: `components/schedule/timeline-view.tsx` — handle swap state
- Modify: `components/schedule/day-view.tsx` — handle swap state
- Modify: `app/s/[id]/schedule-view.tsx` — pass swap handler down

### 5. OG Image

**Design:** Dark background matching brand. Shows:

**Files:**
