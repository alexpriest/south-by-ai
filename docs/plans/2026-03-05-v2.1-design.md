# South by AI v2.1 Design

## Changes from v2

### 1. Date Range Fix

Trim day picker from Mar 8-18 down to **Mar 12-18** (actual conference dates). Remove Mar 8-11 from the `DAYS` constant.

**Files:** `components/quiz/day-picker.tsx`

### 2. Badge Level Selector

New quiz step after name input (step 2 of 6). Single-select from four SXSW badge types:

- **Platinum Badge** ‚Äî "Access to everything: tech, music, film, comedy"
- **Innovation Badge** ‚Äî "Tech sessions, keynotes, networking, exhibitions"
- **Music Badge** ‚Äî "Music showcases, sessions, film screenings, comedy"
- **Film & TV Badge** ‚Äî "Film & TV screenings, sessions, music showcases, comedy"

The selected badge is stored in `QuizState.badge` and used to filter sessions throughout the app. Only sessions whose `badgeTypes` array includes the user's badge are shown ‚Äî in schedule generation, refinement, and browse.

**Data:** Sessions already have `badgeTypes: string[]` from the SXSW API (`primary_credentials` field).

**Files:**
- New: `components/quiz/badge-picker.tsx`
- Modify: `lib/types.ts` ‚Äî add `badge: string` to `QuizState`
- Modify: `components/quiz/quiz-flow.tsx` ‚Äî add badge step, update step count
- Modify: `lib/claude.ts` ‚Äî filter sessions by badge before sending to Claude
- Modify: `app/api/generate/route.ts` ‚Äî pass badge filter through

### 3. Top Picks in Timeline

Update Claude's prompt to return a `priority` field for each session pick:
- `1` = Top pick for this time slot
- `2` = Good alternative
- `3` = Worth considering if you have time

**Timeline layout changes:**
- When sessions overlap, the top pick (priority 1) gets the **full-width block** with a small star icon and "Top Pick" label
- Alternatives (priority 2-3) render as **narrower, faded blocks** (`opacity-70`) beside the top pick
- Non-overlapping sessions render normally regardless of priority

**Prompt changes:** System prompt instructs Claude to assign priority 1 to at most one session per time slot, with alternatives as 2-3.

**Files:**
- Modify: `lib/types.ts` ‚Äî add `priority: number` to `ScheduleSession`
- Modify: `lib/claude.ts` ‚Äî update prompt and response schema to include `priority`
- Modify: `components/schedule/timeline-view.tsx` ‚Äî adjust overlap layout to favor priority 1
- Modify: `components/schedule/timeline-block.tsx` ‚Äî render star badge for priority 1, fade for 2-3

### 4. Timeline Block Popover

Replace the broken inline expand (absolute positioning prevents height growth) with a **click-to-popover**.

On click, a floating card appears anchored to the block showing:
- Session description
- Speakers (as clickable links ‚Äî see section 6)
- AI's reason for picking this session
- Link to SXSW event page (external)

Click outside the popover to dismiss. Popover positions itself above or below the block depending on available space.

**Implementation:** Use a simple state-managed popover with `position: fixed` and a portal, calculated from the block's `getBoundingClientRect()`. No library needed.

**Files:**
- New: `components/schedule/session-popover.tsx`
- Modify: `components/schedule/timeline-block.tsx` ‚Äî replace expand state with popover trigger

### 5. Remove Browse Pages

Delete `/browse` and `/browse/[track]` pages entirely. Replace "Browse All Sessions" link with an external link to `https://schedule.sxsw.com/2026/schedule`.

The SXSW site handles browsing better than we can ‚Äî our value is the personalized AI schedule, not replicating their catalog.

**Files:**
- Delete: `app/browse/page.tsx`
- Delete: `app/browse/[track]/page.tsx`
- Delete: `components/browse/track-card.tsx`
- Delete: `components/browse/session-list.tsx`
- Modify: `app/s/[id]/schedule-view.tsx` ‚Äî change browse link to external SXSW URL
- Modify: `components/schedule/session-card.tsx` ‚Äî remove clickable track link to `/browse`

### 6. Speaker Links

Anywhere speakers appear (popover, list view cards), names become clickable links to `schedule.sxsw.com/2026/search/speaker?q=Speaker+Name`. Opens in new tab.

The SXSW API only returns speaker names (no IDs), so we link to the search page which resolves to their profile.

**Files:**
- New: `lib/speaker-url.ts` ‚Äî `getSpeakerUrl(name: string): string` helper
- Modify: `components/schedule/session-popover.tsx` ‚Äî render speakers as links
- Modify: `components/schedule/session-card.tsx` ‚Äî render speakers as links in list view

### 7. Copy and Cost Fixes

**Homepage copy:** Update "850+ sessions" to dynamically show the actual filtered count (based on badge type if known, otherwise total).

**Prompt optimization:** Before sending sessions to Claude, filter by:
1. Badge type (from quiz)
2. Selected days (already done)
3. Selected interest tracks

This significantly reduces token count and cost, and improves generation speed.

**Files:**
- Modify: `app/page.tsx` or homepage component ‚Äî dynamic session count
- Modify: `lib/claude.ts` ‚Äî add track pre-filter before prompt construction
