# South by AI — Design Document

## Overview

AI-powered SXSW 2026 schedule builder. Users take a quick quiz about their interests, get a personalized day-by-day schedule generated by Claude, refine it via chat, and share it with a unique link that renders a preview card on social media.

**Goal:** Ship before Sunday March 9 to catch peak SXSW planning window (festival starts March 12).

## Stack

- **Frontend:** Next.js (React) + Tailwind CSS
- **Auth:** NextAuth.js — Google + Twitter OAuth
- **AI:** Claude API (Anthropic SDK), called server-side via API routes
- **Storage:** Vercel KV (Redis) for schedules + user data
- **OG Images:** `@vercel/og` (Satori) for shareable schedule cards
- **Deploy:** Vercel
- **Data:** SXSW schedule scraped to static JSON (~850 sessions)

## Pages

### `/` — Landing + Quiz

Hero section with tagline + CTA. Below it, a multi-step quiz:

1. **Interests** (multi-select chips): Tech/AI, Startups, Music, Film, Creator Economy, Design, Culture, Health/Wellness
2. **Vibe** (single select): Deep dives, Rapid-fire (short talks + networking), Mix of both
3. **Days attending** (multi-select): Thu Mar 13 / Fri Mar 14 / Sat Mar 15 / Sun Mar 16
4. **Specific requests** (optional free text): "I heard Amy Webb is good", "defense tech", etc.
5. **Your name**: For the shareable card header

Submit → loading state with rotating copy ("Reviewing 850+ sessions...", "Resolving time conflicts...", "Optimizing your walking route...") → redirect to `/s/[id]`

### `/s/[id]` — Schedule View

Day-by-day timeline showing:
- Session name + time + venue
- One-line AI reasoning ("Why: Amy Webb's trend reports have predicted 94% of major tech shifts")
- Walking context between sessions (venues are scattered across downtown this year — no Convention Center)
- Time conflict warnings if any
- "Refine This" button → chat interface
- "Share" button → copy link + Twitter/LinkedIn share buttons

### `/s/[id]/refine` — Chat Refinement

Chat interface alongside the schedule (sidebar on desktop, below on mobile):
- Schedule stays visible
- User types natural language: "swap the 2pm for something on creator economy", "add more music on Saturday"
- Claude regenerates with schedule context + chat history
- Updates the same schedule ID (shared links always show latest version)

## Auth

NextAuth.js with Google + Twitter OAuth providers.

- Required to generate a schedule
- NOT required to view shared schedules (public URLs)
- User's schedule list tied to their auth session in Vercel KV

## Data Pipeline

1. Scrape schedule.sxsw.com — all sessions with: title, description, track, time, venue, speakers
2. Store as static JSON bundled with the app
3. Claude prompt includes relevant sessions (filtered by day/track) + user preferences
4. Claude returns structured JSON: selected sessions with times, venues, reasoning

## Sharing / Virality

- Each schedule gets a unique URL: `/s/[id]`
- OG image auto-generated via `@vercel/og` showing user's name + schedule highlights
- "Built with South by AI" watermark on share cards
- Share buttons for Twitter and LinkedIn

## Visual Design

Dark mode, festival energy, bold and playful but trustworthy.

**Colors:**
- Background: `#0A0A0A` (near-black)
- Primary: `#FF6B35` (SXSW orange)
- Accent: `#00D4AA` (electric teal)
- Text: `#F5F5F5` (off-white)
- Muted: `#6B7280` (gray)

**Typography:**
- Headings: Space Grotesk (bold grotesque)
- Body: Inter or system sans-serif

**Components:**
- Glassmorphism cards with subtle borders
- Smooth transitions, loading shimmer animations
- Chip selectors for quiz multi-select
- Timeline-style schedule layout

## Storage Schema (Vercel KV)

```
schedule:{id} → {
  userId: string,
  name: string,
  preferences: { interests, vibe, days, freeText },
  schedule: { [day]: Session[] },
  chatHistory: Message[],
  createdAt: timestamp,
  updatedAt: timestamp
}

user:{authId} → {
  scheduleIds: string[],
  name: string,
  provider: "google" | "twitter"
}
```

## Key Risks

- **Scraping reliability:** SXSW schedule site may block or change structure. Mitigation: scrape once, store static JSON, manual fallback.
- **Claude token costs:** Full session list is large. Mitigation: filter by day/track before sending to Claude, use structured output.
- **Timeline:** Must ship by Sunday March 9. Mitigation: cut scope ruthlessly — chat refinement is the stretch goal, quiz → schedule → share is the MVP.
