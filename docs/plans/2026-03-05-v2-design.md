# South by AI v2 Design

## Changes from v1

### 1. Live SXSW Data (replacing mock data)

**Approach:** Fetch from SXSW's internal search API at runtime with ISR caching.

**How it works:**
1. `GET /2026/search/event` to get session cookie + CSRF token
2. `POST /2026/search` with `{models: ["event"], term: "", filters: []}` — returns all ~3,755 events
3. Transform to our Session schema
4. Cache with `unstable_cache` from `next/cache` — revalidate every 15 minutes

**Data mapping** (SXSW API → our schema):
- `_source.name` → `title`
- `_source.event_id` → `id`
- `_source.theme` → `track` (SXSW calls tracks "themes")
- `_source.event_type` → `type` (Session, Screening, Showcase, etc.)
- `_source.format` → `format` (Panel, Solo, Film Screening, Workshop, etc.)
- `_source.date` → `date`
- `_source.start_time` → `startTime` (ISO 8601, convert to local CT)
- `_source.end_time` → `endTime`
- `_source.venue.name` + `_source.venue.root` → `venue`
- `_source.panelists` → `speakers` (array of names)
- `_source.links` → extract tags from label="Tag" entries
- `_source.credential_types` → `badgeTypes`
- `_source.thumbnail_url` → `imageUrl`
- Construct URL: `https://schedule.sxsw.com/2026/events/${event_id}`

**Real SXSW tracks** (the "theme" field):
Tech & AI, Design, Culture, Health, Creator Economy, Brand & Marketing, Cities & Climate, Startups, Startups & Investment, Sports & Gaming, Music, Film & TV, Workplace, Global, Headliner

**Real event types:**
Session, Screening, Showcase, Activation, Comedy Event, Exhibition, Lounge, Networking, Party, Special Event

**Real date range:** March 8-18, 2026 (pre-events start Mar 8, main conference Mar 12-15, music/film through Mar 18)

**Files changed:**
- Delete `data/sessions.json`
- Rewrite `lib/sessions.ts` — live fetch + ISR cache
- Update `lib/types.ts` — add imageUrl, badgeTypes, type fields
- Update `components/quiz/interest-chips.tsx` — use real track names with descriptions

### 2. Calendar/Timeline View

**New schedule visualization** — toggle between List and Timeline views.

**Timeline view:**
- Vertical time axis from 9am to 11pm (Central Time)
- Session blocks positioned by start/end time
- Block height proportional to duration
- Color-coded by track (assign each track a distinct color)
- Click a block → expand to show full details (description, venue, speakers, AI's reason for picking it)
- Show time conflicts visually (overlapping blocks)
- Current time indicator line (during SXSW dates)

**Mobile timeline:**
- Single column, full-width blocks
- Scrollable vertically
- Compact time labels on left edge

**List view** (existing, improved):
- Keep current card-based layout as an alternative
- Add toggle button: "Timeline | List" in the schedule header

**Files:**
- New: `components/schedule/timeline-view.tsx`
- New: `components/schedule/timeline-block.tsx`
- Modified: `app/s/[id]/schedule-view.tsx` — add view toggle

### 3. Category Browsing

**New browse page** at `/browse`.

**Browse index (`/browse`):**
- Grid of track cards
- Each card: track name, session count, track color accent
- Click → `/browse/[track]`

**Track detail (`/browse/[track]`):**
- All sessions in that track, grouped by date
- Filter by event type (Session, Screening, etc.)
- Filter by date
- Each session card has an "Add to Schedule" button (if user has an active schedule)

**Integration with schedule view:**
- Track labels on session cards become clickable → navigate to `/browse/[track]`
- Track color badges throughout the app
